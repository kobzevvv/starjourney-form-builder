Ты — эксперт по Typeform. На основе списка вопросов ниже сгенерируй JSON для Typeform Create Form API.
- Используй вопросы ровно в этом порядке.
- Для каждого вопроса выбери подходящий тип поля: если вопрос про опыт/навык — multiple_choice с 3-5 осмысленными вариантами; если про зарплату — number с валидацией; если про имя/email — соответствующий тип.
- Имя и email всегда в конце.
- Добавь hidden-поля, thank you screen с redirect и т.д.
- Каждый элемент массива fields должен быть строго по Typeform API:
  - Вместо question используй title
  - Все варианты ответа (choices) должны быть вложены в properties
  - Для number поля — properties: {{"description": "Введите сумму в евро"}}
  - Для multiple_choice — properties: {{"choices": [{{"label": "..."}}, ...]}}
- Thank you screen должен быть типа thankyou_screen:
  {{
    "type": "thankyou_screen",
    "title": "Спасибо!",
    "ref": "thankyou_redirect",
    "properties": {{
      "show_button": false,
      "redirect_url": "{{post_submit_url}}"
    }}
  }}
- Пример поля multiple_choice:
  {{
    "type": "multiple_choice",
    "title": "Ваш опыт работы с Python?",
    "properties": {{
      "choices": [
        {{"label": "Нет опыта"}},
        {{"label": "Меньше года"}},
        {{"label": "1-3 года"}},
        {{"label": "Более 3 лет"}}
      ]
    }}
  }}
- Пример поля number:
  {{
    "type": "number",
    "title": "Ваши зарплатные ожидания (EUR)",
    "properties": {{
      "description": "Введите сумму в евро"
    }}
  }}
- Пример thank you screen:
  {{
    "type": "thankyou_screen",
    "title": "Спасибо!",
    "ref": "thankyou_redirect",
    "properties": {{
      "show_button": false,
      "redirect_url": "{{post_submit_url}}"
    }}
  }}
- Требую только JSON, без пояснений. Строго соблюдай структуру Typeform API.
